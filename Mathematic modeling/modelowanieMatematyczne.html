Wstp

Aplikacja Shiny zostaa stworzona w celu umoliwienia interaktywnej analizy regresji liniowej. Regresja liniowa to technika statystyczna wykorzystywana do modelowania zwizku midzy jedn zmienn zalen (Y) a jedn lub kilkoma zmiennymi niezalenymi (X). Celem analizy regresji jest zrozumienie i opisanie tego zwizku, a take prognozowanie wartoci zmiennej zalenej na podstawie zmiennych niezalenych.

Uywane pakiety

library(shiny)
library(ggplot2)
library(nortest)

Dziaanie

Pokaemy teraz na przykadzie, jak dziaa cz “statystyczna” projektu. Najpierw uytkownik wpisuje funkcj, do ktrej zostanie dopasowany model oraz wybiera sigm oraz przedzia na osi X. Dla naszego przykadu przyjm sigm=2 oraz maxX=10. Nasz funkcj natomiast bdzie: \[f(x) = x^2 + 2x + 3\]

set.seed(123)
maxX <- 10 #wybrana przez uytkownika warto x
sigma <- 2 #wybrana przez uytkownika sigma

x <- runif(300, min = 0, max = maxX) 
#generowanie 300 wartoci z rozkadu normalnego

# Definiowanie funkcji kwadratowej
functionInput <- "x^2 + 2*x + 3" #wybrana funkcja
y_true <- eval(parse(text = functionInput)) 
#tworzenie 300 wartoci bdw
error <- rnorm(300, mean = 0, sd = sigma) 
data <- data.frame(x = x, y = y_true + error)

Wygenerowane dane

Dopasowanie modelu

model_formula <- as.formula(paste("y ~ x + I(x^2) + I(x^3) + sin(x) + cos(x)"))
initial_model <- lm(model_formula, data = data)
  
step_result <- step(initial_model)

Powyszy kod uywa funkcji lm(), aby dopasowa do naszych danych funkcj. Teraz zostan pokazane wspczynniki naszego dopasowanego modelu:

Powyszy kod za pomoc testu normalnoci Andersona-Darlinga sprawdza czy reszty naszego modelu s lub nie s zgodne z rozkadem normalnym. Warto wspomnie o zaoeniach testu normalnoci Andersona-Darlinga:

-Dane ilociowe

-Brak zakce

-Nieobcione dane

-Kategorie nieprzekraczalne

-Poziom istotnoci

Teoretyczne Zaoenia A.R.L:

1. Liniowo

Model regresji liniowej zakada, e zaleno midzy zmiennymi niezalenymi a zmienn zalen jest liniowa. To oznacza, e zmiana jednostkowa w zmiennej niezalenej powoduje sta zmian w zmiennej zalenej, niezalenie od jej wartoci. W aplikacji uywany jest model liniowy postaci: y = Ax + Bx^2 +Cx^3 +D^x4 +Esin(x) + Fcos(x) + e, a do dopasowania tego modelu uywana jest funkcja lm():

#initial_model <- lm(model_formula, data = data())

2. Liczba obserwacji

Musi by wiksza bd rwna liczbie parametrw wyprowadzonych z analizy regresji (wspczynniki dla predyktorw, wyraz wolny. W praktyce zakada si, e powinno by co najmniej 15, 20 obserwacji na jedn zmienn w modelu regresji.

3. Homoskedastyczno

Zaoenie o rwnoci wariancji reszt (bdw modelu) dla wszystkich poziomw zmiennych niezalenych. Innymi sowy, oczekujemy, e rozrzut reszt wok dopasowanej linii jest stabilny i nie zaley od wartoci zmiennych niezalenych. NP. dla x^2 + e

4. Brak autokorelacji

Reszty modelu s niezalene od siebie. Innymi sowy, brak jest wzorca w pozostaociach, nie wystpuje zaleno pomidzy resztami dla rnych obserwacji.

acf(residuals(step_result))

5. Reszty maj rozkad zbliony do rozkadu normalnego

Normalno rozkadu reszt moemy oszacowa korzystajc z testw normalnoci, takich jak: test Shapiro-Wilka, test Komogorowa-Smirnowa. My natomiast uywamy testu Andersona-Darlinga.

qqnorm(residuals(step_result))

Interfejs Uytkownika (UI)

Wygld

-RadioButtons (“sigma”): Pozwala uytkownikowi wybra warto parametru sigma, ktry wpywa na poziom szumu w danych.

-TextInput (“functionInput”): Pozwala uytkownikowi wprowadzi funkcj f(x), ktra reprezentuje relacj midzy zmiennymi.

-SliderInput (“maxX”): Pozwala uytkownikowi ustawi maksymaln warto x w generowanych danych.

-CheckboxInputs (“includeSin”, “includeCos”): Pozwalaj uytkownikowi uwzgldni dodatkowe skadniki funkcji sinus i cosinus w modelu.

-ActionButton (“fitButton”): Uruchamia proces dopasowywania modelu po klikniciu.
